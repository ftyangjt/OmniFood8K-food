# 代码使用指南

本文档说明如何使用当前仓库完成环境检查、深度图生成、训练、测试和自定义图片推理。

所有命令默认在项目根目录运行。

```powershell
cd C:\Users\lhxsy\OneDrive\DeepLearning\OmniFood8K-food
conda activate omnifood
```

## 1. 主要入口文件

真实脚本集中在 `scripts/` 目录：

```text
scripts/train_nutrition.py       训练入口
scripts/test.py                  数据集测试入口
scripts/infer_nutrition.py       单张图片或文件夹推理入口
scripts/run.py                   通用 Depth Anything V2 深度图生成入口
scripts/generate_8k_depth.py     批量生成 OmniFood8K 的 rgb-d.png
utils/datasets.py                数据集读取定义
utils/utils_data222.py           DataLoader 创建逻辑
```

根目录不再保留脚本入口。请统一使用 `scripts/` 下的真实脚本。

## 2. 环境检查

```powershell
python --version
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.version.cuda)"
python -c "import torch, torchvision, timm, transformers, cv2, open3d, pytorch_wavelets; print('deps ok')"
```

正常情况下应看到 Python 3.10、PyTorch CUDA 11.8，并输出：

```text
deps ok
```

## 3. 权重文件

下载或准备以下预训练权重，并放入 `pth/`：

```text
pth/swin_base_patch4_window12_384_22k.pth
pth/convnext_small_22k_1k_384.pth
pth/depth_anything_v2_vitl.pth
```

训练得到的最优权重建议保存到：

```text
trained_weights/
```

例如：

```text
trained_weights/omnifood8k/ckpt_best.pth
```

## 4. 准备 OmniFood8K 数据

当前仓库默认使用：

```text
data/0-OminiFood8k/
```

期望结构：

```text
data/0-OminiFood8k/
  train_new333.txt
  test_new333.txt
  8036/
    <sample_id>/
      camera_4.jpg
      rgb-d.png
```

如果还没有 `rgb-d.png`，先批量生成：

```powershell
python scripts\generate_8k_depth.py --data-root .\data\0-OminiFood8k --encoder vitl --ckpt .\pth\depth_anything_v2_vitl.pth
```

## 5. 训练

推荐命令：

```powershell
python scripts\train_nutrition.py --dataset nutrition8K --data_root_8k .\data\0-OminiFood8k --b 2 --epoch 150 --log .\logs\omnifood8k --save_dir .\trained_weights
```

训练日志保存到：

```text
logs/omnifood8k/train_log.txt
```

最优权重保存到：

```text
trained_weights/omnifood8k/ckpt_best.pth
```

如果只是确认流程能否跑通，可以先做 1 个 epoch 的冒烟训练：

```powershell
python scripts\train_nutrition.py --dataset nutrition8K --data_root_8k .\data\0-OminiFood8k --b 2 --epoch 1 --log .\logs\omnifood8k_smoke --save_dir .\trained_weights
```

Windows 下默认 `--num_workers 0`，避免 DataLoader 多进程递归启动问题。流程稳定后可以尝试：

```powershell
python scripts\train_nutrition.py --dataset nutrition8K --data_root_8k .\data\0-OminiFood8k --b 2 --epoch 150 --log .\logs\omnifood8k --save_dir .\trained_weights --num_workers 4
```

## 6. 测试

使用训练得到的最优权重：

```powershell
python scripts\test.py --dataset nutrition8K --data_root_8k .\data\0-OminiFood8k --b 8 --ckpt .\trained_weights\omnifood8k\ckpt_best.pth
```

测试脚本会输出 Calories、Mass、Fat、Carb、Protein 的 MAE 和 PMAE，以及 Mean MAE、Mean PMAE、Sum PMAE。

注意：论文中的 PMAE 通常以百分数展示，代码输出的是小数。例如 `0.086` 对应 `8.6%`。

## 7. 自定义图片推理

检测单张图片：

```powershell
python scripts\infer_nutrition.py --img-path .\your_food.jpg --ckpt .\trained_weights\omnifood8k\ckpt_best.pth --depth-ckpt .\pth\depth_anything_v2_vitl.pth --save-depth
```

检测一个文件夹：

```powershell
python scripts\infer_nutrition.py --img-path D:\your_images --ckpt .\trained_weights\omnifood8k\ckpt_best.pth --depth-ckpt .\pth\depth_anything_v2_vitl.pth --save-depth
```

输出结果保存到：

```text
outputs/infer_nutrition/predictions.csv
```

## 8. 常见问题

### 缺少 rgb-d.png

确认每个样本目录中都有：

```text
camera_4.jpg
rgb-d.png
```

如果缺少深度图，运行：

```powershell
python scripts\generate_8k_depth.py --data-root .\data\0-OminiFood8k --encoder vitl --ckpt .\pth\depth_anything_v2_vitl.pth
```

### DataLoader 在 Windows 上报错

当前脚本默认：

```text
--num_workers 0
```

不要在 Windows 上直接使用很大的 `num_workers`，否则可能触发 multiprocessing spawn 报错。

### 测试结果明显差于论文

优先检查三点：

```text
1. checkpoint 是否真的是 OmniFood8K 对应权重
2. rgb-d.png 的生成方式是否和训练时一致
3. train_new333.txt/test_new333.txt 的字段顺序是否匹配代码
```

当前 `nutrition8K` 字段顺序为：

```text
sample_id mass calories protein fat carb
```
