# 项目思路及流程

本文档从代码角度解释 OmniFood8K-food 的整体思路、数据流、模型结构和训练流程。

## 1. 项目目标

项目目标是从单张食物图片估计 5 个连续营养指标：

```text
calories  热量
mass      质量
fat       脂肪
carb      碳水化合物
protein   蛋白质
```

它不是分类任务，而是多任务回归任务。

整体流程：

```text
RGB 食物图像
  -> 预测深度图
  -> RGB 分支提取颜色、纹理、语义特征
  -> 深度分支提取形状、结构、体积相关特征
  -> 多模态融合
  -> 输出 5 个营养数值
```

## 4. 训练入口

训练入口：

```text

```

启动后主要流程：

```text
1. 解析命令行参数
2. 设置数据路径、权重路径、日志路径
3. 设置随机种子
4. 构建模型
```

## 5. 模型结构

当前模型是双分支结构：

```text
RGB 分支：
RGB image -> SwinTransformer -> 多尺度 RGB 特征

深度分支：
Depth image -> DepthAdapterV4 -> ConvNeXt -> 多尺度深度特征

融合部分：
RGB 多尺度特征 + Depth 多尺度特征 -> FusionNet_3Branch_UNet_FFT

预测部分：
融合特征 -> 5 个预测头 -> 5 个营养数值
```

代码中对应模块：

```python
net = SwinTransformer()
net2 = convnext_small()
adapter = DepthAdapterV4()
net_cat = FusionNet_3Branch_UNet_FFT()

pre_net1 = FeatureFusionNetwork222_Mask()
pre_net2 = FeatureFusionNetwork222_Mask()
pre_net3 = FeatureFusionNetwork222_Mask()
pre_net4 = FeatureFusionNetwork222_Mask()
pre_net5 = FeatureFusionNetwork222_Mask()
```

五个预测头分别负责：

```text
pre_net1 -> calories
pre_net2 -> mass
pre_net3 -> fat
pre_net4 -> carb
pre_net5 -> protein
```

## 6. RGB 分支

RGB 图像输入 Swin Transformer：

```python
output_rgb = net(inputs)
r0, r1, r2, r3, r4 = output_rgb
```

训练中主要使用：

```text
r1, r2, r3, r4
```

这些特征包含纹理、颜色、食物区域和高层语义信息。

## 7. 深度分支

深度图先经过 `DepthAdapterV4`：

```python
inputs_rgbd = adapter(inputs_rgbd)
```

它用于校正预测深度图的尺度和局部结构。Depth Anything V2 生成的是相对深度，不一定具有真实物理尺度，因此需要一个可学习的适配模块。

然后送入 ConvNeXt：

```python
output_hha = net2(inputs_rgbd)
d1, d2, d3, d4 = output_hha
```

深度分支主要提供食物高度、形状、体积感、盘中占比和空间结构。

## 8. 多模态融合

融合模块位于：

```text
model/dual_swin_convnext.py
```

当前使用：

```python
FusionNet_3Branch_UNet_FFT
```

核心思想是把特征拆到频域中处理：

```text
低频：整体轮廓、区域大小、形状、体积趋势
高频：边缘、纹理、细节、食物表面颗粒
```

RGB 擅长颜色和纹理，Depth 擅长几何和空间结构。频域融合可以让模型更清楚地区分并融合这些信息。

融合后得到多尺度特征：

```python
o1, o2, o3, o4
```

这些特征再送入五个预测头。

## 9. 损失函数

基础损失是 L1Loss：

```python
criterion = nn.L1Loss()
```

每个营养指标的损失会做归一化：

```python
batch_size * L1(pred, target) / target.sum()
```

这样可以避免数值大的指标支配训练。例如 calories 通常比 fat 大很多，如果直接相加，模型会更偏向优化 calories。

此外还有跨模态对齐损失：

```python
loss_align = inter_modal_alignment_loss(o1) * 0.1
```

最终总损失大致为：

```python
loss = k1 * calories_loss
     + k2 * mass_loss
     + k3 * fat_loss
     + k4 * carb_loss
     + k5 * protein_loss
     + loss_align
```

其中 `k1` 到 `k5` 来自动态任务权重模块。

## 10. 动态任务权重

动态任务权重位于：

```text
model/three_D.py
```

模块名：

```python
DynamicTaskPrioritization
```

它会根据当前各任务损失更新权重。误差较大的任务会被适当提高权重，避免五个营养指标训练不均衡。

## 11. 保存 checkpoint

每个 epoch 后会执行一次测试：

```python
train(epoch, net)
test(epoch, net)
scheduler.step()
```

测试时计算五个指标的 PMAE，并求和：

```python
loss_pmae = calories_pmae + mass_pmae + fat_pmae + carb_pmae + protein_pmae
```

如果当前 `loss_pmae` 比历史最好更低，就保存最优模型：

```text
trained_weights/<log目录名>/ckpt_best.pth
```

保存内容包括 Swin 分支、ConvNeXt 分支、DepthAdapter、融合网络、五个预测头、优化器状态和 epoch。

## 12. 测试流程

测试入口：

```text
scripts/test.py
```

测试命令：

```powershell
python scripts\test.py --dataset nutrition8K --data_root_8k .\data\0-OminiFood8k --b 8 --ckpt .\trained_weights\omnifood8k\ckpt_best.pth
```

测试流程：

```text
1. 构建同样的模型结构
2. 加载 ckpt_best.pth
3. 读取测试集
4. RGB 进入 Swin
5. Depth 进入 Adapter + ConvNeXt
6. 多尺度特征融合
7. 五个预测头输出
8. 统计 MAE 和 PMAE
```

## 13. 一句话总结

项目主线可以概括为：

```text
先用 Depth Anything V2 给 RGB 食物图生成深度图，
再用 Swin 提取 RGB 颜色、纹理、语义特征，
用 ConvNeXt 提取深度几何、结构、体积特征，
通过 FFT 频域融合对齐 RGB 和 Depth 信息，
最后用五个独立回归头预测热量、质量、脂肪、碳水和蛋白质。
```

核心点：

```text
1. RGB + Depth 双模态营养估计
2. 频域高低频融合
3. 五任务动态加权回归
```
