我看了一圈，当前模型大致是：RGB 用 Swin，深度/伪深度用 ConvNeXt，经 `FusionNet_3Branch_UNet_FFT` 做低层频域融合、高层拼接，再用 5 个独立 `FeatureFusionNetwork222_Mask` 分别回归 Calories/Mass/Fat/Carb/Protein。

**最值得优先改的点**

1. **修正模态对齐损失**
   [train_nutrition.py](</c:/Users/lhxsy/OneDrive/DeepLearning/OmniFood8K-food/scripts/train_nutrition.py:259>) 里 `loss_align = inter_modal_alignment_loss(o1)`，但 `o1` 来自 [dual_swin_convnext.py](</c:/Users/lhxsy/OneDrive/DeepLearning/OmniFood8K-food/model/dual_swin_convnext.py:69>) 的 `torch.cat([fused_high, fused_low])`，已经不是 RGB/Depth 两半了。现在这个 loss 实际在对齐“高频/低频”，不是 RGB/Depth。建议把 alignment 放在融合前的 `r1/d1` 或高层 `cat3/cat4` 里真正 RGB/Depth 分开的特征上。

2. **把 5 个重复预测头改成共享主干 + 5 维联合输出**
   当前 [train_nutrition.py](</c:/Users/lhxsy/OneDrive/DeepLearning/OmniFood8K-food/scripts/train_nutrition.py:110>) 建了 5 套完整 `FeatureFusionNetwork222_Mask`，而每个 head 都在重复做卷积、cross-attention、pooling。营养指标高度相关，尤其 `calories ≈ 4*protein + 4*carb + 9*fat`。建议改为一个共享 fusion head，最后输出 `[B, 5]`，再加轻量 task-specific adapter 或 task embedding。

3. **预测值加正值约束或 log-space 回归**
   现在最后是 [fusion.py](</c:/Users/lhxsy/OneDrive/DeepLearning/OmniFood8K-food/modules/fusion.py:108>) 的 `Linear(1024, 1)`，理论上会输出负营养值。建议最后用 `Softplus`，或预测 `log1p(label)` 再 `expm1` 还原。营养回归通常 label 长尾明显，这个改动经常比堆模块更稳。

4. **频域融合应从固定相加改为可学习门控**
   [MultiFreqFusion](</c:/Users/lhxsy/OneDrive/DeepLearning/OmniFood8K-food/model/dual_swin_convnext.py:16>) 现在用固定 `freq_threshold=0.1`，然后 `r_high + d_high`、`r_low + d_low`。这太硬了：不同食物、不同尺度需要不同频率权重。建议加入可学习 gate，例如：
   `fused_high = gate_h * r_high + (1-gate_h) * d_high`，低频同理；也可以让 threshold 或频段权重可学习。

5. **高层融合太粗糙**
   低两层用了 FFT 融合，但 [cat3/cat4](</c:/Users/lhxsy/OneDrive/DeepLearning/OmniFood8K-food/model/dual_swin_convnext.py:108>) 只是 RGB/Depth concat。高层语义对“食物种类”和“份量估计”很关键，建议高层也用 cross-attention / gated fusion / FiLM，而不是裸拼接。

6. **深度分支需要更强的模态适配**
   [DepthAdapterV4](</c:/Users/lhxsy/OneDrive/DeepLearning/OmniFood8K-food/modules/adapter.py:6>) 只是全局 `alpha/beta` + 小 CNN 残差。由于 ConvNeXt 是 RGB 预训练，直接吃深度图会有域偏移。建议：
   - depth 输入单独标准化，不沿用 RGB ImageNet norm；
   - adapter 残差最后一层零初始化，让训练从接近 identity 开始；
   - 或把 ConvNeXt stem 改成专门的 depth stem，再接预训练主干。

7. **任务权重策略有明显问题**
   [train_nutrition.py](</c:/Users/lhxsy/OneDrive/DeepLearning/OmniFood8K-food/scripts/train_nutrition.py:308>) 的 `current_kpis` 第三项重复用了 `mass_loss`，没有用 `fat_loss`。这会让 fat 任务权重更新偏掉。模型层面建议用已有的 `AutomaticWeightedLoss`，或更标准的 uncertainty weighting / GradNorm。

8. **引入营养一致性约束**
   这是这个任务很适合加的结构先验：让模型同时预测 5 个指标，但在 loss 中加入：
   `calories_pred` 与 `4*protein + 4*carb + 9*fat` 的一致性约束。  
   不一定强制完全相等，因为食物标签可能有误差，但可以作为 soft regularization。

**我的优先级建议**

先做：修 alignment loss、修 `fat_loss` 权重 bug、把输出改成正值/log-space。  
再做：5 个 head 合并成共享多任务 head。  
最后做：可学习频域融合和更强高层 RGB-D cross-attention。

这几个里，前两项属于“现在可能在误导模型”的问题，收益/风险比最高。
